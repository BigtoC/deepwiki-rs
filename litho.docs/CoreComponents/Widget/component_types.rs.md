# Component Types 组件技术文档

## 1. 组件概述

### 1.1 主要功能和作用
`component_types.rs` 是一个UI组件类型管理组件，主要负责定义和管理UI组件的类型信息。它提供了组件类型的映射功能，管理组件元数据，支持组件分类和识别，并提供组件类型相关信息。

### 1.2 在系统中的位置和重要性
该组件位于系统的UI层，是UI组件管理的核心部分。它的重要性评分为0.80，表明其在系统中具有重要地位。该组件不依赖于其他组件，但可能被其他UI相关组件所依赖。

## 4. 接口说明

### 4.1 对外提供的接口
- `display_name`: 获取组件类型的显示名称
- `folder_name`: 获取组件类型的文件夹名称
- `description`: 获取组件类型的描述
- `fmt`: 格式化组件类型信息
- `default`: 获取默认组件类型
- `map_by_path_and_name`: 根据路径和名称映射组件类型
- `ComponentTypeMapper`: 组件类型映射器

### 4.2 输入输出参数
- `display_name`: 无输入，输出显示名称
- `folder_name`: 无输入，输出文件夹名称
- `description`: 无输入，输出描述
- `fmt`: 输入格式化字符串，输出格式化后的组件类型信息
- `default`: 无输入，输出默认组件类型
- `map_by_path_and_name`: 输入路径和名称，输出映射的组件类型
- `ComponentTypeMapper`: 无输入，输出组件类型映射器

### 4.3 调用示例
```rust
// 获取组件类型的显示名称
let name = ComponentType::Button.display_name();

// 根据路径和名称映射组件类型
let component_type = ComponentType::map_by_path_and_name("path/to/component", "Button");
```
